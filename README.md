# internal-tool-frontend

## 👨‍🔧 프로젝트 개요

이 저장소는 시설관리 도메인(설비, 소방, 에너지 관리)을 기반으로 한
내부 업무용 웹 도구를 직접 설계·구현하는 학습/실전 프로젝트입니다.

단순 튜토리얼 코드가 아니라, 실제 시설관리 현장에서 사용하는
점검/대시보드 흐름을 반영하는 것을 목표로 합니다.

## 📚 진행 방식 (BLOCK 단위)

이 프로젝트는 **"내부툴 개발자 ver.3" 커리큘럼**에 따라
BLOCK 단위(예: 004, 005, …, 012…)로 꾸준히 확장하는 방식으로 진행 중입니다.

- 각 BLOCK은 2~3일 분량의 학습·구현 단위입니다.
- BLOCK마다 아래 순서를 반복합니다.
  1. 목표 기능 정의 (예: 로그인 검증, 점검 등록, 목록 렌더링)
  2. HTML/CSS/JS 설계
  3. 구현 및 디버깅 (브라우저/콘솔 확인)
  4. 불필요 코드 정리
  5. `git commit` → `git push`

> 같은 파일을 계속 수정하지만,  
> **Git 커밋 로그와 코드 주석으로 BLOCK 경계가 남도록 관리**하고 있습니다.

## ✅ 현재 진행 상황 (BLOCK 기준)

- 001: 점검 등록 폼 HTML 구조
- 목적: 내부툴 프론트엔드 기초부터 누적 기록

- BLOCK 001–004
  - 기본 레이아웃, 폼 구조, 입력값 추출, 기본 검증

- BLOCK 005–007
  - 로그인 페이지(auth.html)
  - AND/OR 조건 검증
  - 권한/작업 조건 가드 패턴 기초

- BLOCK 008–009
  - 작업 가능 여부를 판단하는 `canUserStartWork` 순수 함수 도입
  - 사용자 상태 스냅샷(`userState`) 개념 정리

- BLOCK 010–012
  - 점검 등록 폼(index.html) 유효성 검사
  - 점검 데이터를 `localStorage`에 저장
  - 점검 목록(checks.html) 렌더링 및 선택 상태(`.active`) 구현
  - 선택된 점검 상세 패널, 대시보드 준비

- BLOCK 013
  - 점검 상세 패널 정리
  - 대시보드 통계 카드(총 건수, 상태별 집계) 연결
- BLOCK 014 : 대시보드 요약 카드
  - `localStorage`에 저장된 점검 데이터를 불러와 총 건수/정상/이상 건수 집계
  - 메인 대시보드에서 한눈에 설비 상태를 확인할 수 있는 통계 카드 UI 구성

- BLOCK 015 : 점검 목록 검색/필터
  - 설비명 텍스트 검색 + 상태별 필터(전체/정상/이상/점검중) 기능 추가
  - `getFilteredChecks()`로 검색/필터 로직을 한 곳에서 관리하도록 구조화

- BLOCK 016 : 점검 목록 정렬 옵션
  - 날짜 최신순/오래된순, 설비명 가나다순 정렬 옵션 추가
  - `sortOrder` 상태를 기준으로 `Array.sort()`로 정렬 처리

- BLOCK 017 : 조건에 맞는 점검 개수 표시
  - 현재 검색/필터/정렬 조건에 맞는 점검 건수를 상단에 표시
  - 사용자 입장에서 “지금 보고 있는 목록이 몇 건인지” 직관적으로 확인 가능

- BLOCK 018 : 상태 배지 스타일링
  - 상태 텍스트를 `정상/이상/점검중`에 따라 색상이 다른 배지 형태로 표시
  - `getStatusClass()` 함수로 상태 텍스트 → CSS 클래스 매핑 분리

- BLOCK 019 : 점검 목록 페이지네이션 도입
  - `currentPage`, `PAGE_SIZE`를 도입해 한 페이지당 N개씩 잘라 보여주도록 리팩토링
  - 검색/필터/정렬 결과에 따라 페이지 수 자동 계산 및 이전/다음 버튼으로 이동 가능

- BLOCK 020 : 점검 데이터 초기화 & 필터 영역 UX 개선
  - “점검 데이터 초기화” 버튼으로 예시 데이터/사용자 데이터 재설정 기능 추가
  - 현재 조건에 맞는 점검 개수 표시 + 필터/정렬/초기화 버튼 레이아웃 정리
