# internal-tool-frontend

## 👨‍🔧 프로젝트 개요

이 저장소는 시설관리 도메인(설비, 소방, 에너지 관리)을 기반으로 한
내부 업무용 웹 도구를 직접 설계·구현하는 학습/실전 프로젝트입니다.

단순 튜토리얼 코드가 아니라, 실제 시설관리 현장에서 사용하는
점검/대시보드 흐름을 반영하는 것을 목표로 합니다.

## 📚 진행 방식 (BLOCK 단위)

이 프로젝트는 **"내부툴 개발자 ver.3" 커리큘럼**에 따라
BLOCK 단위(예: 004, 005, …, 012…)로 꾸준히 확장하는 방식으로 진행 중입니다.

- 각 BLOCK은 2~3일 분량의 학습·구현 단위입니다.
- BLOCK마다 아래 순서를 반복합니다.
  1. 목표 기능 정의 (예: 로그인 검증, 점검 등록, 목록 렌더링)
  2. HTML/CSS/JS 설계
  3. 구현 및 디버깅 (브라우저/콘솔 확인)
  4. 불필요 코드 정리
  5. `git commit` → `git push`

> 같은 파일을 계속 수정하지만,  
> **Git 커밋 로그와 코드 주석으로 BLOCK 경계가 남도록 관리**하고 있습니다.

## ✅ 현재 진행 상황 (BLOCK 기준)

- 001: 점검 등록 폼 HTML 구조
- 목적: 내부툴 프론트엔드 기초부터 누적 기록

- BLOCK 001–004
  - 기본 레이아웃, 폼 구조, 입력값 추출, 기본 검증

- BLOCK 005–007
  - 로그인 페이지(auth.html)
  - AND/OR 조건 검증
  - 권한/작업 조건 가드 패턴 기초

- BLOCK 008–009
  - 작업 가능 여부를 판단하는 `canUserStartWork` 순수 함수 도입
  - 사용자 상태 스냅샷(`userState`) 개념 정리

- BLOCK 010–012
  - 점검 등록 폼(index.html) 유효성 검사
  - 점검 데이터를 `localStorage`에 저장
  - 점검 목록(checks.html) 렌더링 및 선택 상태(`.active`) 구현
  - 선택된 점검 상세 패널, 대시보드 준비

- (진행 중) BLOCK 013+
  - 점검 상세 패널 정리
  - 대시보드 통계 카드(총 건수, 상태별 집계) 연결
